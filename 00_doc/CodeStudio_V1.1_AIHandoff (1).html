COMPONENT REFERENCE
#Component役割 / 仕様
1 HeaderBar 最上部バー。ロゴ・SearchBox・MoreMenu を内包。
拡張予定 将来の機能追加時に HeaderBar 内に モード切り替えボタン（例：Edit / Preview / Schema）を追加できるスロットを確保。ボタンは activeMode state で管理。
2 SearchBox Ctrl+F トリガーのファイル内検索UI。
仕様追加 検索ヒット後、十字キー（↑↓）で次の一致単語へジャンプ。Monaco の findController.moveToNextMatch() / moveToPreviousMatch() を呼び出す。ヒット数バッジ「3 / 12」表示。
3 MoreMenu 右上「⋮」ボタン。クリックで画面全体にオーバーレイするハンバーガーパネルを表示（position:fixed / backdrop-blur）。
メニュー項目：スキーマ設定 / ルール設定 / ショートカット一覧 / フォルダ構成 Template（新規追加）
Template追加 Template 選択画面でプロジェクト構成テンプレート（Novel / Game / Script等）を選び、フォルダを一括生成。
4LeftIconNav左端 Activity Bar。アクティブ項目に左ボーダー。モバイル時は非表示（BottomNav に移行）。
5 FileSidebar ファイルツリー領域。FileItem / FileFolder を内包。
仕様追加 A 境界線（ResizeHandle）：右端ドラッグで幅を 120px〜400px の範囲で自由調整。幅は useLayoutStore.sidebarWidth に保存。
仕様追加 B ヘッダー右寄せアイコン：① 📁 フォルダ新規作成　② 📄 ファイル新規作成　③ 🗂 Template（MoreMenu[3]のフォルダ構成テンプレートを直接起動）
6 TabBar ファイルタブ行。アクティブタブに上ボーダーアクセント。
Memo連携 行番号 ↔ MemoEditor の関連づけ仕様：
① LineNumbers（⑦）の行をクリック → selectedLine を useMemoStore に書込み
② MemoPanel が自動で「filePath:activeFile.path + line:selectedLine」を表示
③ 同一ファイルの別タブに切り替えると selectedLine はリセット
④ MemoItem クリック時は TabBar が対応ファイルのタブをアクティブに切り替えてから Monaco がスクロール
7LineNumbers行番号列。クリック→ useMemoStore.setSelectedLine(n) 発火。メモ付き行には左端に ● インジケーター表示。
8MonacoEditorWrapperMonaco Editor 本体 wrap。JSON優先・カーソル取得・executeEdits for Word挿入・deltaDecorations for Memoハイライト。
9 FAB 要検討 エディタ右下フローティングボタン。現状「新規セグメント追加」予定だが、コンテキストメニュー方式への変更も検討中（右クリック or 長押しで操作パレット表示）。実装フェーズで確定する。
10WordListPanelキーワードボタン。クリック → Monaco executeEdits 挿入 / ドラッグ → DnD 挿入。KEY / ATTR / VOID / LIST カテゴリ色分け。
11 RightIconNav W/F/?/M パネル切替。未対応メモ数バッジ M(n)。
拡張設計 パネルは プラグイン方式で追加可能に設計：
PANEL_REGISTRY = [ {id:'word', icon:'W', component:WordListPanel}, ... ]
新パネルは registry に追加するだけで自動的にアイコンと切替が追加される。
12FooterStatusBarBranch・Sync・行列番号・エンコード・通知を常時表示。各ステータスはクリックでアクション（行番号→移動ダイアログ等）。

04ZUSTAND STORES

State Design

Zustand 4ストア設計 → Core Features → Memo Feature

📐 useLayoutStore
// State isMobile: boolean leftSidebarOpen: boolean rightSidebarOpen: boolean activeRightPanel: 'word'|'func'|'manual'|'memo' sidebarWidth: number // ⑤ ResizeHandle activeMode: 'edit'|'preview'|'schema' // ① 拡張用 // Actions setMobile(bool) toggleLeft() toggleRight() setActivePanel(panel) setSidebarWidth(px) setActiveMode(mode)
レイアウト全体の見た目・開閉状態を管理。
モバイル判定、左右サイドバーの開閉、右パネルの表示切替を一元管理する。
sidebarWidth はリサイズハンドルで変更、activeMode はHeaderBarの切り替えボタンと連動。
✏ useEditorStore
// State openFiles: FileEntry[] activeFile: FileEntry | null cursorPosition: { line, column } isDirty: boolean searchQuery: string // ② 検索文字列 searchIndex: number // ② 現在のヒット番号 // Actions openFile(entry) closeFile(path) setActive(path) setCursor(line, col) setDirty(bool) setSearchQuery(q) nextMatch() // ② 十字キー→次のヒット prevMatch() // ② 十字キー→前のヒット
エディタで開いているファイルとカーソル状態を管理。
タブで複数ファイルを同時に開ける。検索クエリとヒット番号も管理し、十字キーで次のマッチに飛ぶ機能と連動する。isDirty は未保存の変更フラグ。
📁 useProjectStore
// State rootHandle: FileSystemDirectoryHandle // File System Access API fileTree: TreeNode[] isLoaded: boolean // Actions selectRoot() // showDirectoryPicker refreshTree() createFile(path) // ⑤ ヘッダーアイコン createFolder(path) // ⑤ ヘッダーアイコン applyTemplate(tpl) // ③ Template適用 saveFile(path, content)
ローカルフォルダとファイルツリーを管理。
File System Access API を通じてユーザーのローカルフォルダに直接アクセスする。createFile / createFolder はFileSidebarヘッダーのアイコンから呼ばれる。applyTemplate はMoreMenuのTemplate選択から呼ばれ、フォルダ構成を一括生成する。
💬 useMemoStore
// State memos: Memo[] activeMemo: Memo | null selectedLine: number | null filterMode: 'all'|'unresolved' // Actions addMemo(filePath, line, content) editMemo(id, content) deleteMemo(id) toggleResolved(id) jumpToMemo(id) // ⑥ Monaco scroll連携 setSelectedLine(n) // ⑦ LineNumbers click setFilterMode(mode)
行ベースのコメント（Memo）を管理。
行番号クリック → setSelectedLine → MemoPanel が自動で該当行・ファイルパスを表示。jumpToMemo は対応ファイルをアクティブにして Monaco をスクロールさせる。filterMode で未対応のみ/全件 を切替。データは IndexedDB に永続化。
